The Client-Server Communication Process

When clients connect to the server, the server must accept the connection. When the client sends a command and type of the related command to the server, the arguments depending on the type if it has one, the server must parse it, do the appropriate process of the command, send back the client a response about the success of the command, and depending whether or not, the server must also send back the type of the message, and the arguments of that type.

When the server communicates with the client after the client is connected, the server must send back the client the response, it's type and its arguments if it has and close the connection to the client. It happens on every command sent from the client.

The lifecycle of a command:

  1. The client connects to the server.
  2. The server acknowledges the client and accepts the connection.
  3. When the client sends a command and type, the server then must parse it and do the appropriate process of the command then send back it the response.
  4. Optionally depending on the command, the server must send back the type of the message and the arguments of that type.
  5. Finally, close the connection of the client.
